Kanban

Kanban - japoński termin określający listę przeczy do zrobienia. Ułatwia śledzenie postępów pracy. Składa się z trzech obszarów:

  • to do
  • doing
  • done

Praktyczny kanban online: Trello.com

Nazewnictwo w programowaniu:

  • bez polskich znaków
  • bez spacji
  • po angielsku
  • konsekwentnie, według jakiegoś schematu.
  • nie zaczynamy od cyfry

Konwencja nazewnictwa

  • camelCase - np. myHeader
  • kebab-case - np. my-header
  • snake_case - np. my_header
  • PascalCase - np. MyHeader

Różne rodzaje stylowania w CSS

  • styl inline - wewnątrz znacznika, miejscowo.
    <znacznik style="color:red;">treść</znacznik>
  • atrybut class - dla grupy elementów.
    <znacznik class="name">treść</znacznik>
    w CSS klasę oznaczamy kropką: .name { } Nazwa klasy w kebab-case.
  • stylowanie za pomocą id
    <znacznik id="name">treść</znacznik>
    w CSS identyfikator oznaczamy krzyżykiem (hashem): #name { } Na stronie tylko jeden znacznik powinien mieć id="name" - służy do wewnętrznej nawigacji po stronie.
  • !important - polecenie wstawiane po wartości danej właściwości
    selektor {
    właściwość: wartość !important;
    }
    To zła praktyka. To polecenie unieważnia i nadpisuje inne reguły, powodując zamieszanie - lepiej nie używać.

Specyficzność w CSS (od najważniejszej):

  1. !important
  2. style inline
  3. id
  4. class
  5. element

Jeśli specyficzność jest równa, ważniejszy jest selektor "bliżej" danego elementu (czyli te facto to, co jest niżej w kodzie).

Jak działa internet

HTTP(S) - Hypertext Transfer Protocol (Secure) - protokół, którego używa przeglądaka do transportowania HTML-a. Pobiera i wyświetla pliki znajdujące się w stronach internetowych.

URL - Uniform Resource Locator - adres strony. Ten adres składa się z kilku części:

https:// cotenfrontend :443 / index.html ?parametry=wartość

  • https:// - protokół
  • cotenfrontend.pl - host (adres serwera)
  • :443 - port (ta część zazwyczaj jest dla użytkownika niewidoczna). Wskazuje przeglądarce, na którym porcie na serwerze znajdują się pliki strony
  • / - bardzo ważny znak / - root - przy podawaniu ścieżki do zasobów (src). Jeśli podajemy źródło ze slashem na początku (src="/img.jpg"), przeglądarka szuka w katalogu głównym. Jeśli podajemy bez (src="img.jpg") - szuka w katalogu aktualnym, czyli np w katalogu podstrony. Dlatego lepiej podawać źródło (na serwerze) ze slashem na początku.
  • index.html - ścieżka dostępu do zasobu
  • ?paramert=wartość - parametry

DNS - Domain Name System - system tłumaczący nazwę mnemoniczną (czyli "dla ludzi") na odpowiadający jej adres IP.

parsowanie - sprawdzanie składni

Kody odpowiedzi protokołu HTTP:

  • 10x - czekaj
  • 200 - wszystko ok.
  • 30x - Przekierowania: szukasz w złym miejscu - przekieruję cię w dobre, bo wiem, gdzie.
  • 404 - nie mam tego, czego potrzebujesz :(
  • 50x - błędy serwera

Kody możemy sprawdzić w Devtools (F12) w zakładce Network.

Życie w pudełku

Właściwość display może mieć różne wartości:

  • display:block - kontener na całą szerokość strony, domyślny dla wielu znaczników
  • display:inline - treść ustawia się w linii, szerokość własna, nie można nadawać innej width i height (domyślny np. dla <a>
  • display:inline-block - łączy właściwości obu. Kontenerowi można nadać width i height (oraz min/max-width/height)

overflow - odcinanie treści, która wystaje

  • overflow:hidden - nadmiar treści ukryty
  • overflow:visible - nadmiar treści widoczny
  • overflow:auto - przy nadmiarze treści pojaiwa się scroll w elemencie

Jednostki:

  • 0 - może być podawane bez jednostki
  • px - piksele
  • % - procenty (np width: 50% to 50% szerokości body albo rodzica)
  • vw - viewport width (dotyczy zawsze wymiarów okna)
  • vh- viewport height

Box model

Box model zawsze jest prostokątem, który potem może być modyfikowany.

  • padding - wewnętrzny margines
  • border - granica (kontur) elementu. Można zapisywać łącznie (border: 3px solid red;). Border-radius - zaokrągla narożniki (50%=kółko)
  • margin - zewnętrzny margines. Marginesy sumują się. "margin: auto" wyśrodkowuje elementy o właściwości "display: block", jeśli mają określoną szerokość.

Właściwość box-sizing (domyślna) - bierze pod uwagę wymiary content-box. Właściwość border-box bierze pod uwagę wymiary całego elementu z padding i konturem.

Position

  • position: static (domyślna) - element ma swoją stałą pozycję na stronie
  • position: relative - podobnie, ale można określić wymiary i z-index. Daje możliwość użycia "absolute" w Środku
  • position: absolute - stała względem rodzica. Ustalana za pomocą top, left, right, bottom (pozycja lewego górnego rogu), z-index ("głębokość")
  • position: fixed - element "przyklejony" do okna w ustalonej pozycji
  • position: sticky - porusza się jak "fixed", ale w ramach większego elementu, a nie całego okna - "dokleja" się do treści powyżej i poniżej

Float - własność do opływania elementów. Może przyjmować wartości: right, left, none, inherit.

Clear - "anti-float", czięki tej właściwości element nie opływa innego, tylko ustawia się pod spodem.

JavaScript - zmienne

Zmienna w języku JS nie jest wartością, tylko wskazuje wartość.
Rodzaje zmiennych:

  • let - możemy w jej ramach podstawiać różne wartości
  • const - stałe połączenie z jedną wartością
  • var - wychodzi z użycia

Nzawy zmiennych podajemy w camelCase. Zmienne mogą przyjmować różne wartości, np. tekst lub liczby.
Typy proste (primitive values):

  • String - tekst (w ' ' lub - rzadziej - " ")
  • Number - liczba
  • Bigint - duża liczba, większa od 253
  • boolean - true/false
  • null - celowo niezdefiniowana
  • undefined - błąd; jeśli wcześniej niezdefiniowana
  • *symbol - unikalne opakowanie na obiekty

Polecenie typeof - sprawdza typ naszej zmiennej

Immutability - "niezmienność" - nie możemy zmieniać wartości typów prostych. To, co zmieniamy, to połączenie między nimi a zmiennymi.

DOM - Document Object Model - to nie kod, ale już wykonana instrukcja - czyli to, co widać w dev-tools. JS korzysta z DOM i w ten sposób możemy się ze stroną komunikować. Robimy to za pomocą polecenia:

document.querySelector('.selector') - polecenie umożliwiające zmiany na stronie. Selektor oznaczamy w kodzie HTML jako dodatkową klasę, ale dla porządku dodajemy zawsze --js (.selector--js). Ta klasa nie służy do stylowania, tylko jest odnośnikiem dla JS.

JS - obiekt

Obiekt jest strukturą danych. Może zawierać wiele własności (property - zmienne) lub metod (method - funkcje). W konsoli są one wyświetlane alfabetycznie. Własność dzieli się na:

klucz: wartość;

Klucz musi być unikalny w ramach jednego obiektu. Wartość może być prymitywem, funkcją lub zagnieżdżeniem

Przykład:


const car = {
type: 'passenger car',
mark: 'Peugeot',
model: '207',
yearOfManufacture: 2005,
navigation: (destination) => {
console.log(`Going to: ${destination}`)
},
engine: 1.6,
fuel: 'oil',
}

Sposoby wyświetlania jednej z własności:

▪ dot notation - np. console.log(car.mark), console.log(car.navigation('Warszawa')

[ ] bracket notation - np. console.log(car['type'])

Reset i Normalizacja

Przeglądarka posiada swoje domyślne style, które nadpisujemy w naszym CSS. Reset je unieważnia. Trzeba być ostrożnym w jego używaniu - strona staje się nieczytelna i traci na dostępności. NIE WOLNO resetować outline! (outline: 0 lub none) - wtedy linki nie są podświetlane przy pomocy klawisza Tab (dla niedowidzących i osób posługujących się tylko klawiaturą).

Normalizacja - tworzymy specjalny plik normalize.css (link do treści tutaj), automatycznie wyrównujący drobne różnice między przeglądarkami. Najlepiej zawsze dodawać go w części <head> kodu przed naszym arkuszem stylów.

Warto też ustawić w całym html box-sizing na border-box (a dla potomków: inherit). 


html {
box-sizing: border-box;
}
                
*, *:before, *:after {
box-sizing: inherit;
}

JS-logika

Operatory logiczne zazwyczaj są używane z wartościami typu Boolean (true/false). Operatory porównania to:

  • >
  • <
  • <=
  • >=
  • == - obie strony równe (pojedyńcze "=" to operator przypisania) - np. 2=='2' - true
  • === - obie strony równe, sprawdza typeof - np. 2==='2' - false (bo number != string)
  • != - obie strony różne
  • !== - obie strony różne i sprawdza typeof

Porównania można stosować na zmiennych i na obiektach.

False ma 6 wartości:

  • false
  • 0
  • ","",``(pusty string)
  • null
  • undefined
  • NaN - not a number

True to wszystkie pozostałe rzeczy.

Łączenie warunków:

  • AND - && (oba warunki muszą być true)
  • OR - || (jeden z nich musi być true) 
if((myVariable == 2) || (otherVariable !== 'test')) {...}

NOT - ! - odwraca wartość wyrażenia logicznego; zaprzeczenie, lepiej unikać bo komplikuje kod!

Warunek IF () {...} ELSE {...}
else if - zdefiniowany (może być powtarzany)

Instrukcja SWITCH - rospatruje różne warianty w zależności od przypadku: 


      switch (myVariable) {
        case 7:
        ...
        break;
        case 9:
        ...
        break;
        default:
        ... ;
      }

Zapis skrócony: - ? - np: 

(25 > 20) ? console.log('to prawda') : console.log('to nieprawda');

Formularze

Formularze służą do komunikacji z użytkownikiem.

<form> - opakowanie na formularz

<label> - opis pola. <label for="idName"> - po nakliknięciu aktywuje pole do wpisywania.

<input> - pole formularza, miejsce do wpisywania. Według dokumentacji input opakowujemy w <p> i dodajemy mu id dla label.
<p><input name="inputName" id=idName">

Atrybuty inputa:

  • number (przyjmuje tylko cyfry)
  • tel (przyjmuje nr telefonu)
  • email (przyjmuje e-mail, sprawdza format, pokazuje podpowiedzi z przeglądarki
  • range (suwak) - trzeba ustawić min, max, step
  • password (zasłania to, co wpisujemy)
  • file (do załączenia pliku)
  • submit (wysyła cały formularz) - to, co jest w value="", wyświetli się na guziku

<fieldset> - grupa pól
<legend> - opis tej grupy pól

<select> - daje nam liste do wyboru
<option> - wartości listy. Dajemy value=""

<textarea> - pole do wprowadzenia dłuższego tekstu

Wysyłanie formularza: atrybut action="https://jakasstronaodbiorcza.pl". To, co jest wysyłanie to nie opis, a value!

Żądanie typu GET - domyślne, wrzuca zawartość formularza do adresu - lepszy do pobierania.
Żądanie typu POST - zostawia czysty adres, lepsze (bezpieczniejsze) do wysyłania formularza.

enctype - określa typ formularza. Domyślny to anctype="application/x-www-forum-urlencoded". Jeśli wysyłamy z formularza plik, musimy go zmienić na enctype="multipart/form-data".

Właściwość "required" - ustawia pole wymagane.
Właściwość "pattern" - pozwala ustawić format wprowadzanej treści (np. kod pocztowy). Wzory na html5pattern.com

CSS Media Queries

RWD - Responsive Web Design - strona przystosowana do urządzeń o różnej wielkości ekranu.

@media screen / print / all / .. - zapytanie o media.

W media queries używamy operatorów logicznych and, , (przecinek, odpowiadający "or") i not 


@media screen and (min-width: 400px) and (max-width: 600px) {
  /tutaj selektory dla danej wielkości ekranu
  (okna przeglądarki)/
}

Niektóre elementy można "chować" za pomocą display: none.

Breakpoint - miejsce "łamania" strony, przeskoku do stylów dla innej wielkości.

Wiele osób poleca zasadę Mobile First, czyli pierw definiowanie stylów które obowiązują na całej stronie, łącznie z wersją mobilna, a następnie dokładamy breakpointy i zmieniamy konkretne własności elementów by dopasować je do większych ekranów.

Pseudoelementy & pseudoklasy

Pseudoelementy - oznaczane są za pomocą ::. Jedne z najpopularniejszych to ::before i ::after. Pozwalają dodać treść przed i po elemencie bez dopisywania jej w kodzie HTML. - za pomocą właściwości content: "dodana treść". Treść ta w prkatyce jest używana głównie jako element dekoracyjny, np jako kropka przed tytułem.

Pseudoselektory - oznaczane są za pomocą :. Wprowadzają modyfikacje wynikające z innego działania, np :hover, :active. :visited czy :focus.

  • :first-child
  • :last-child
  • :nth-child
  • :not(selektor) - zaprzeczenie

Pseudoklasy mogą być zagnieżdżane. Więcej informacji w oficjalnej dokumentacji.

Node.js

Node.js jest to środowisko umożliwiające uruchomienie JavaScript poza przeglądarką (terminal). Można w nim uruchomić pojedynczy plik *js, niepodpięty pod HTML. Node.js nie ma dostępu do DOM. Może byc używany także w backendzie.

Aby uruchomić w terminalu należy wpisać: node nazwapliku.js.

npm jest to repozytorium z gotowymi "paczkami" (modułami) do node'a, przyśpieszają i ułatwiają one wiele rzeczy. Zawiera gotowe biblioteki i narzędzie aktualizowane na bieżąco przez twórców. Jest to coś na zasadzie "google play" czy "apple store" dla frontendowców. Inicjuje się go w katalogu za pomocą polecenia npm init.

Po instalacji pierwszej paczki tworzy się katalog node_modules, który zawiera "składniki" danej paczki. Katalog ten jest bardzo "ciężki" i nie commitujemy go, korzystamy z pliku .gitignore. Informacje o zainstalowanych paczkach znajdują się w pliku package.json. Nawet gdy usuniemy katalog node_modules, dzięki temu plikowi możemy bez problemu pobrać znów wszystkied potrzebne elementy za pomocą npm install.

Sass

Sass jest to preprocesor CSS. Posiada on własną składnię oraz składnię CSS, z której będziemy korzystać. Pliki *.sass są zamieniane przez Sass w *.css, który jest zrozumiały dla przeglądarki.

$ - zmienna, która pozwala zdefiniować własność często powtarzającą się w dokumencie.

@mixin - funkcja łącząca kilka właściwości; można potem zmieniać jej parametry. Wywołuje się ją za pomocą @include. Polecenie to pozwala automatycznie dodać jej zawartość do danego elementu lub klasy.

@extend - grupuje (w pliku .css) selektory z podanymi właściwościami.

& - parent selector - znacznie skraca kod, pozwala zastępować długie nazwy BEM-a, można zagnieżdżać, jest to też bardziej czytelne, np: 


      .test {

        &:hover {
          color: blue;
        }  
        &__item {
          background-color: green;
          &--special {
            background-color: orange;
          }  
        }
      }

Gdy korzystamy z preprocesorów, EDYTUJEMY tylko pliki .scss, ponieważ nadpisują pliki .css. NIE EDYTUJEMY wygenerowanych plików .css!

Moduły

Moduły porządkują projekt, skracją kod, umożliwiają korzystanie z gotowych elementów.

W SCSS można zaimportować moduł z innego pliku (@import "źródło) lub bezpośrednio z internetu (bez pobierania) - np. czcionkę.

Moduły w JS: w przeglądarce działają bez pobierania modułu ES6 Modules. Możemy eksportować i importować moduły między własnymi plikami. aby tu umożliwić, do znacznika script dodajemy type="module". 


<script type="module" scr="main.js"></script>

Eksport nazwany: w pliku, z którego chcemy eksportować moduł, dodajemy przed funkcją polecenie "eksport", np: 

export const hello = (name) => {...}

Wywołujemy moduł poleceniem "import". Możemy importować kilka rzeczy na raz: 

import {hello, myFunction} from './file.js';

Eskport domyślny: przez polecenie "export default". Najpierw definiujemy zmienną lub funkcję, a następnie piszemy polecenie: 

export default hello;

Wywołujemy ją natomiast pod dowolną nazwą oraz bez nawiasów: 

import greeting from './file.js';

Dzięki importowi nie trzeba w wielu miejscach kopiować tego samego kodu, wystarczy wywołać funkcję.

Można importować również z node modules: 

import nazwaRzeczy from 'nazwa-modułu';

W przypadku gdy mamy długą lub kolidującą z czymś nazwę, możemy ją zamienić: 

import {veryLongName as newName} from 'ścieżka-do-pliku';

Możemy importować całą zawartośc pliku: 

import * as newName from 'ścieżka-do-pliku';

W importowaniu oraz ogarnianiu innych podobnych procesów pomaga nam WEBPACK..

Git Branche

Brachne (gałęzie) pozwalają ne bezkarne testowanie bez ruszania oryginału. Służą do bardziej efektywnego programowania oraz pozwalają zrównolegić pracę, czyli kilka osób może robić różne rzeczy i na końcu się je łączy w brachnu głównym. Dzięki temu zwiększa się czytelność postępu prac nad projektem.

Branch master jest główną gałęzią. Polecenie git branch podaje nam info na jakiej gałęzi jesteśmy. Git branch nazwabrancha tworzy nową gałąź o nazwie "nazwabrancha". Za pomocą git checkout "nazwabrancha" przęłączamy się między gałęziami. Gdy jesteśmy na pobocznych branchach, commity nie wchodzą na mastera.

Polecenie git push --set-upstream origin "nazwabrancha" wypycha nam commita tworząc od razu "nazwabrancha" na Githubie.

Aby połączyć gałęzie: git merge "nazwabrancha" z poziomu mastera.

Jest to język znaczników. Nie posiada semantyki. Jest czytelny zarówno jako źródło, jak i wyniku. Posiada jedno uniwersalne formatowanie tekstu i bezpieczne wprowadzanie treści bez konieczności korzystania z HTML-a. Nie trzeba także pisać edytora.

JS Array (tablice)

Tablice są strukturą danych.

Elementami tablicy mogą być napisy, stringi, liczby, wartości true/false, obiekty itd. Można miksować typy danych.

Zapis:

const myArray = ['element0', 'element1', 'element2']

Index - pozycja elementu w tablicy. Tablice indexujemy od zera. Aby pobrać dany element: tablica[index], np.: myArray[1] wywoła drugi element.

myArray.length podaje ilość elementów (od 1!). To samo dotyczy stringów - można wypisać dowolną literę wyrazu lub sprawdzić ilość liter. wypycha nam commita tworząc od razu "nazwabrancha" na Githubie.

myArray.push('element') - dodaje kolejny element na końcu tablicy.

myArray.unshift('element') - dodaje element na początku tablicy.

myArray.shift('element') - usuwa pierwszy element (indeksy pozostałych się zmieniają).

myArray.pop('element') - usuwa ostatni element.

myArray.indexOf('element') - podaje indeks danego elementu.

myArray.splice(index, liczbaElementów) - usuwa n elementów zaczynając od pozycji index.

myArray.slice(index, stop) - pobiera n elementów zaczynając od pozycji index i kończąc na pozycji stop (ale już bez tej pozycji, zatrzymuje się przed nią). Przykład: myArray.slice(2, 3) - pobierze tylko element 2.

JS Loops (pętle)

Pętla mówi, ile razy lub do którego momentu polecenie (blok kodu) ma się powtarzać.

Podstawowa pętla w JS to FOR:

for (inicjalizacja; warunek stopu; wyrażenie inkrementacji) {
    kod
}

i to licznik pętli

i++ zwiększa wartość i o 1 po wykonaniu kodu funkcji (postinkrementacja)

i-- zmniejsza wartość i o 1 po wykonaniu kodu funkcji (podekrementacja)

++i zwiększa wartość i o 1 przed wykonaniu kodu funkcji (preinkrementacja)

--i zmniejsza wartość i o 1 przed wykonaniu kodu funkcji (predekrementacja)

Za pomocą pętli for możemy np wypisać elementy jakiejś tablicy: 

for (let i=0; i <  myArray.lenght; i++ {
    console.log (`pod indeksem ${i} jest element ${myArray[i]}`)
}

Gdy będziemy dodawać jakieś elementy do tablicy, będą one również wypisane.

Kolejna pętla to WHILE:

whlie (warunek logiczny) {
    kod
}

Jeżeli nie warunek logiczny nie będzie się zmieniał, będzie stała wartość, pętla będzie działać w nieskończoność - doporowadzić to może do zabicia przeglądarki.

Pętla DO WHILE - pierw wykonuje kod, następnie sprawdza warunek - kod wykona się co najmniej raz. 

do {
    kod
} while (warunek logiczny)

Pętla FOR IN - przebiega przez własności obiektu: 

for (let myProperty in myObject) {
    console.log(myObject[myProperty]);
}

Pętla FOR OF - przebiega przez elementy tablicy: 

for (let element of myArray) {
    console.log(element);
}

Animacje i przejścia

Tutaj lista właściwości, które mogą być animowane: Animatable CSS properties.

transition: name duration timing-function delay; - polecenie przejścia. Gdy zmianie mają ulec tylko 2 własności, wymieniamy je po przecinku. W devtoolsach możemy edytować efekt (timing-function - krzywa do wyginania).

will-change: name; - przekazuje przeglądarce, że jakaś własność będzie zmienialna animacją. Pomaga niby przeglądarce, że animacja działa szybciej, nie jest często używane.

transform: translate (x, y); - przesuwa element.

transform: rotate (xdeg); - obraca element o daną ilość stopni. Warto dodać własność np. transform-origin: left center; - ustala z którego punktu transformacja ma się wykonać.

transform: scale (x); - skaluje element. Również można podać z którego punktu jak w rotate.

Transformacje te nie przesuwają innych elementów na stronie, warto z nich korzystać.

Animacje - posiadają więcej opcji niż transition. Mogą trwać niezależnie czy dajemy jakiś hover czy nie, mogą działać bez interakcji. Definiujemy za pomocą @keyframes, podobnie jak w SCSS np: 

@keyframes name {
    from {
      transform: scale (1);
    }
    to {
      transform: scale (2);
    }
} 
.className {
    animation: name duration timing-function delay iteration-count etc.

WAI / WCAG / A11y / ARIA - czyli odpowiedzialność w internecie

WAI - Web Accessibility Initative. Inicjatywa która co jakiś czas sporządza poradniki, wytyczne, jak nasze strony powinny się zachowywać niezależnie od naszej sprawności.

WCAG - Web Content Accessibility Guidelines. Dokument opisujący jak nasza strona powinna się zachowywać, działać aby osoby niepełnosprawne mogły bez problemu z niej korzystać. Niektóre strony np. instytucji, mają specjalne wymogi.

The A11y (Accessibility) Project - projekt do pomocy w tworzeniu dostępych stron. Więcej info na: a11yproject.com/.

Fundacja Widzialni - zajmują się tym tematem w Polsce. Więcej info na: widzialni.org.

Rady:

  2. Trzymaj kontrast - można sprawdzic tu: color.a11y.com/Contrast/
  3. Dobieraj dobrze kolory - colorsafe.co/ Jak coś zwracać uwagę projektantom.
  4. Unikaj autoplay - nie straszyć ludzi dzwiękami z nienacka!
  5. Nie ograniczaj czasowo rzeczy które mają jakąś treść - można niezdążyć.
  6. Nie usuwaj :focus i outline - można stylować.
  7. Używaj alt="" i kończ opisy kropką. Jak są obrazki, opisywać dokładnie.
  8. ARIA - Accessible Rich Internet Applications - wytyczne, atrybuty wzbogacające naszą stronę o dostepność. Nie wpływają na wygląd, ale pomagają niepełnosprawnym. w3.org/WAI/standards-guidelines/aria/
